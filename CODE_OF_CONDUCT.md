# Code of Conduct

IroHarness is for people building character macro harnesses, agent adapters,
avatar bodies, and realtime companion systems together.

## Expected Behavior

- Be direct, specific, and respectful.
- Critique ideas and implementations, not people.
- Assume maintainers will ask for tests or contract updates when behavior
  changes.
- Keep discussions focused on identity boundaries, adapter contracts,
  permissions, realtime behavior, and Project OS state.

## Unacceptable Behavior

- Harassment, threats, insults, or discriminatory language.
- Publishing private credentials, personal data, private logs, or unreleased
  security details.
- Repeatedly derailing issues or pull requests after maintainers have asked to
  move the discussion.
- Pressuring contributors to expose private character prompts, private memory,
  or production user data.

## Reporting

For security issues, follow [SECURITY.md](./SECURITY.md).

For conduct issues, open a private maintainer contact path when one is
available. If there is no private path yet, open a minimal public issue asking
for maintainer contact without sharing private details.

Maintainers may edit, hide, lock, or remove comments and contributions that
violate this code.
