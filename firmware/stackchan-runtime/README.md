# IroHarness StackChan Runtime

This is the IroHarness-owned StackChan firmware runtime. It is derived from
AIAvatarStackChan and keeps the same useful device-side shape:

- SD-card `/config.json`
- Wi-Fi profile selection
- WebSocket connection to the host
- microphone upload
- text, audio, touch, and vision invoke
- speaker playback
- face, mouth, LED, and StackChan motion control

The runtime connects to the IroHarness trusted StackChan gateway. It does not
own character identity, memory, Project OS, LLM provider routing, STT
credentials, TTS credentials, Codex OAuth, or work delegation permissions.

## Build

Use the basic PlatformIO project:

```bash
cd firmware/stackchan-runtime/examples/basic
pio run
pio run --target upload
pio run --target uploadfs
```

The data image contains:

```text
data/config.json
data/avatar/*.png
```

Before flashing, replace `data/config.json` with the values generated by:

```bash
iroharness connect stackchan ~/.iroharness/apps/iroha --host-url http://MAC_MINI_IP:4182
```

The generated firmware config lives at:

```text
~/.iroharness/apps/iroha/.iroharness/connections/stackchan-firmware-config.json
```

Copy these fields into `data/config.json`:

- `wifi_networks`
- `ws_host`
- `ws_port`
- `ws_path`
- `user_id`
- `channel`
- microphone, playback, display, PTT, touch, and vision settings

Do not commit real Wi-Fi passwords or device tokens.

## Audio Codec Note

The firmware can play raw PCM directly. Encoded audio, including WAV from
AivisSpeech, needs either a firmware playback converter or host-side conversion
to PCM before the `chunk` message is sent. Until that is finalized, use the
mock TTS path or a PCM-capable TTS/relay when validating speaker playback.
